API Reference

Complete reference for all API endpoints available in Doctor Care SaaS platform, including subscription management, super admin, and multi-tenant features.

Base URL

Base URL: http://localhost:3000/api

All API endpoints are prefixed with /api

Most endpoints require authentication via NextAuth.js session.

Authentication

NextAuth Session Management

POST /api/auth/[...nextauth]

NextAuth.js handles all authentication routes automatically.

Create Admin User

POST /api/auth/create-admin

Creates an initial admin user for system setup.

Request Body:
{
  "name": "Admin Name",
  "email": "admin@example.com",
  "password": "secure_password"
}

Patients

Get Patients

GET /api/patients

Retrieve paginated list of patients with search capability.

Query Parameters:

  • page - Page number (default: 1)
  • limit - Items per page (default: 10)
  • search - Search query

Create Patient

POST /api/patients
Request Body:
{
  "name": "Patient Name",
  "email": "patient@example.com",
  "contactNumber": "+1234567890",
  "address": "Patient Address",
  "dateOfBirth": "1990-01-01",
  "gender": "Male"
}

Get Single Patient

GET /api/patients/[id]
PUT /api/patients/[id]
DELETE /api/patients/[id]

Get, update, or delete a specific patient by ID.

Response Formats

Success Response

{
  "success": true,
  "data": { ... },
  "message": "Operation completed successfully"
}

Error Response

{
  "error": "Error message",
  "details": "Additional error details",
  "status": 400
}

Paginated Response

{
  "data": [...],
  "pagination": {
    "total": 100,
    "page": 1,
    "limit": 10,
    "totalPages": 10
  }
}

HTTP Status Codes

Success Codes

  • 200 OK - Request successful
  • 201 Created - Resource created

Error Codes

  • 400 Bad Request
  • 401 Unauthorized
  • 404 Not Found
  • 500 Internal Server Error
