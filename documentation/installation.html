Installation Guide

Follow this step-by-step guide to install and set up Doctor Care SaaS Platform with complete subscription management, multi-tenancy, and super admin features.

Prerequisites: Make sure you have Node.js (18.0+), MongoDB, and a Stripe account (for SaaS features) before proceeding.

1 Download and Extract Files

After purchasing from CodeCanyon, you'll receive a download package.

Step 1.1: Extract main_files.zip

Locate the downloaded file and extract it:

# Extract the main files
unzip main_files.zip

Step 1.2: Extract doctor-care-next.zip

Inside the main_files folder, you'll find doctor-care-next.zip:

# Navigate to main files and extract the application
cd main_files
unzip doctor-care-next.zip

Step 1.3: Navigate to Project Folder

# Enter the project directory
cd doctor-care-next

2 Install Dependencies

Install all required Node.js packages using your preferred package manager:

Using npm

npm install

Using yarn

yarn install

Using pnpm

pnpm install

This process may take a few minutes depending on your internet connection.

3 Environment Configuration

Configure your environment variables for database connection and authentication:

Step 3.1: Create Environment File

# Copy the example environment file
cp .env.example .env.local

Step 3.2: Configure Environment Variables

Edit the .env.local file with your settings:

# Database Configuration
MONGODB_URI=your_mongodb_connection_string_here
# NextAuth Configuration
NEXTAUTH_SECRET=your_nextauth_secret_key_here
NEXTAUTH_URL=http://localhost:3000
# Environment
NODE_ENV=development

Important Security Note

Never commit your .env.local file to version control. It contains sensitive credentials.

4 Database Setup

You can use either a local MongoDB installation or MongoDB Atlas (cloud database):

Local MongoDB

If you have MongoDB installed locally:

MONGODB_URI=mongodb://localhost:27017/doctor-care

MongoDB Atlas (Recommended)

For cloud database:

MONGODB_URI=mongodb+srv://username:password@cluster.mongodb.net/doctor-care
Getting MongoDB Atlas (Free):
  1. 1. Visit MongoDB Atlas
  2. 2. Create a free account and cluster
  3. 3. Create a database user and get connection string
  4. 4. Replace the connection string in your .env.local file

5 Seed Database (Optional)

Populate your database with sample medications data to get started quickly:

Seed Medications Database

Run the medications seeding script:

node scripts/medications-seed.js

This script will populate your database with a comprehensive medication library containing common drugs and their details.

6 SaaS Platform Setup

Configure the SaaS features including Stripe payments, subscription plans, and email system:

Stripe Integration Setup

Step 6.1: Create Stripe Account

  1. 1. Visit stripe.com and create an account
  2. 2. Complete your business verification
  3. 3. Navigate to Developers ‚Üí API Keys

Step 6.2: Get API Keys (Configure via Super Admin Panel)

Stripe credentials are configured through the Super Admin settings panel, not environment variables.

  1. 1. Copy your Stripe API keys from the dashboard
  2. 2. After creating super admin, login and go to Settings
  3. 3. Navigate to "Payment Gateway" tab
  4. 4. Enter your Stripe credentials in the admin panel

Step 6.3: Configure Webhooks

  1. 1. In Stripe Dashboard ‚Üí Developers ‚Üí Webhooks
  2. 2. Add endpoint: https://yourdomain.com/api/webhooks/stripe
  3. 3. Select events: customer.subscription.*, invoice.*

Subscription Plans Setup

Step 6.4: Seed Subscription Plans

# Seed default subscription plans
node scripts/seed-subscription-plans.js

This creates default plans: Basic ($29/month), Professional ($79/month), Enterprise ($149/month)

Email System Configuration (Optional)

Step 6.5: Email configuration is managed through the Super Admin panel (recommended)

Email Configuration: Configure email settings through the Super Admin panel after login.

  1. 1. Login as super admin
  2. 2. Navigate to Settings ‚Üí Email tab
  3. 3. Choose email provider (Gmail, SMTP, SendGrid, etc.)
  4. 4. Enter credentials and test email functionality

Email is optional during initial setup. You can configure it later via the admin panel.

7 Create Super Admin User

Create a super admin user to manage the SaaS platform, subscription plans, and global settings:

Create Admin User

After starting the server, make a GET request to:

# GET request to create default admin
GET /api/auth/create-admin

Default Super Admin Credentials:
Email: superadmin@dms.com
Password: 12345
Role: Super Admin (platform management access)

SECURITY WARNING: Change these default credentials immediately after first login! Go to Profile ‚Üí Change Password to set a strong, unique password.

You can access this endpoint directly in your browser at http://localhost:3000/api/auth/create-admin after starting the server.

8 Start Development Server

Start the development server to begin using Doctor Care SaaS platform:

Demo Mode (Optional)

Enable demo mode to show demo login buttons with pre-configured credentials:

# Start with demo mode enabled
NEXT_PUBLIC_DEMO_MODE=demo npm run dev

Demo credentials:
Super Admin: superadmin@dms.com / 12345
Practice Admin: admin@dms.com / 12345

Using npm

npm run dev

Using yarn

yarn dev

Using pnpm

pnpm dev

Success!

Open http://localhost:3000 in your browser to access Doctor Care.

Production Deployment

For production deployment, you have several options:

Vercel (Recommended)

Easy deployment with Vercel:

# Build the application
npm run build
# Deploy to Vercel
npx vercel

Self-Hosted

Build and start production server:

# Build for production
npm run build
# Start production server
npm start
