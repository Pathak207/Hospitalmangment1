Configuration Guide

Learn how to configure and customize Doctor Care for your practice needs.

Environment Variables

Configure these environment variables in your .env.local file:

Database Configuration

# MongoDB Connection String
MONGODB_URI=mongodb://localhost:27017/doctor-care
# For MongoDB Atlas:
MONGODB_URI=mongodb+srv://username:password@cluster.mongodb.net/doctor-care

Required for database connection. Use local MongoDB or MongoDB Atlas cloud database.

Authentication Configuration

# NextAuth Secret Key
NEXTAUTH_SECRET=your_secret_key_here
# Application URL
NEXTAUTH_URL=http://localhost:3000

Generate Secret: Use openssl rand -base64 32 to generate a secure secret key.

Environment Settings

# Environment Mode
NODE_ENV=development
# For production:
NODE_ENV=production

SaaS Platform Configuration

Note: Stripe credentials are configured through the Super Admin panel, not environment variables.

Login as super admin → Settings → Payment Gateway tab to configure Stripe integration.

# Demo Mode (Optional)
NEXT_PUBLIC_DEMO_MODE=demo
# Email Configuration (Configure via Super Admin Panel)
# Email settings: Set via Super Admin → Settings → Email tab
# Supports Gmail, SMTP, SendGrid, Mailgun providers

SaaS-specific environment variables. Email and Stripe settings are configured through the Super Admin panel.

Application Settings

Configure these settings through the application's Settings page after installation:

Date Format Options

  • MM/DD/YYYY US Format (12/25/2024)
  • DD/MM/YYYY European Format (25/12/2024)
  • YYYY-MM-DD ISO Format (2024-12-25)
  • MMM DD, YYYY Long Format (Dec 25, 2024)

Currency Options

  • USD US Dollar ($)
  • EUR Euro (€)
  • GBP British Pound (£)
  • CAD Canadian Dollar (C$)
  • AUD Australian Dollar (A$)
  • INR Indian Rupee (₹)

Settings are stored in localStorage and automatically synchronized across browser tabs.

Appointment Configuration

Configure appointment scheduling settings through the time slots API:

Default Appointment Settings:
{
  "startTime": "08:30 AM",
  "endTime": "05:00 PM",
  "interval": 30,
  "workDays": [1, 2, 3, 4, 5],
  "lunchBreak": {
    "start": "12:30 PM",
    "end": "01:30 PM",
    "enabled": true
  }
}

Time Configuration

  • startTime: Practice opening time
  • endTime: Practice closing time
  • interval: Appointment duration in minutes

Working Days

  • workDays: Array of working days
  • 0: Sunday
  • 1-5: Monday to Friday
  • 6: Saturday

Database Configuration

MongoDB Connection Options

Local MongoDB

mongodb://localhost:27017/doctor-care
  • • Install MongoDB locally
  • • Default port 27017
  • • No authentication required

MongoDB Atlas

mongodb+srv://user:pass@cluster.mongodb.net/doctor-care
  • • Cloud-hosted database
  • • Free tier available
  • • Automatic backups

Database Seeding

The system includes medication seeding scripts:

# Seed medications database
npm run seed
# Force overwrite existing data
npm run seed:force
